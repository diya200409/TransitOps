# 🚀 Deploy TransitOps NOW - 15 Minute Guide

**Quick deployment guide to get TransitOps live in 15 minutes!**

---

## 🎯 Fastest Path (Recommended)

### **Backend: Render** ✅ Free PostgreSQL included
### **Frontend: Vercel** ✅ Automatic deployments

Total cost: **$0/month** (with limitations)

---

## Step 1: Deploy Backend (5 minutes)

### **1.1 Sign Up for Render**
- Go to: https://render.com/register
- Sign up with GitHub

### **1.2 Create New Blueprint**
1. Click **"New +"** button
2. Select **"Blueprint"**
3. Connect your GitHub account (if not connected)
4. Select **`diya200409/TransitOps`** repository
5. Click **"Connect"**

Render will automatically:
- Detect `render.yaml`
- Create PostgreSQL database
- Create web service
- Build and deploy!

### **1.3 Wait for Deployment**
- Watch the build logs (5-8 minutes)
- Once deployed, click on your service
- Copy the URL: `https://transitops-api-XXXX.onrender.com`

### **1.4 Seed Database**
1. In Render dashboard, click your backend service
2. Click **"Shell"** tab (left sidebar)
3. Run these commands:
```bash
cd transitops-backend
python seed.py
```

✅ **Backend is LIVE!**

---

## Step 2: Deploy Frontend (3 minutes)

### **2.1 Sign Up for Vercel**
- Go to: https://vercel.com/signup
- Sign up with GitHub

### **2.2 Import Project**
1. Click **"Add New..."** → **"Project"**
2. Select **`TransitOps`** repository
3. Configure build settings:
   - **Framework Preset**: Vite
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`

### **2.3 Add Environment Variable**
1. Scroll down to **"Environment Variables"**
2. Add:
   ```
   Name: VITE_API_BASE_URL
   Value: https://transitops-api-XXXX.onrender.com
   ```
   (Use YOUR backend URL from Step 1.3)

3. Click **"Deploy"**

### **2.4 Wait for Build**
- Watch build logs (2-3 minutes)
- Once deployed, Vercel will show your URL
- Example: `https://transitops-XXXX.vercel.app`

✅ **Frontend is LIVE!**

---

## Step 3: Connect Frontend & Backend (2 minutes)

### **3.1 Update Backend CORS**
1. Go back to Render dashboard
2. Click your backend service
3. Go to **"Environment"** tab
4. Click **"Add Environment Variable"**
5. Add:
   ```
   Name: TRANSITOPS_CORS_ORIGINS
   Value: https://transitops-XXXX.vercel.app
   ```
   (Use YOUR Vercel URL)

6. Click **"Save Changes"**
7. Service will automatically redeploy (~1 minute)

✅ **CORS is configured!**

---

## Step 4: Test Your Live App! (1 minute)

1. Visit your Vercel URL: `https://transitops-XXXX.vercel.app`
2. You should see the login page
3. Login with:
   ```
   Email: fleetmanager@transitops.com
   Password: password123
   ```

🎉 **You're LIVE!**

## ⚡ Pro Tips

### **1. Free Tier Limitations**

**Render Free Tier:**
- ❌ Backend sleeps after 15 minutes of inactivity
- ⏱️ First request after sleep: 30-60 seconds to wake
- ✅ Wakes up automatically on request

**Solution:**
- Upgrade to Render Starter ($7/mo) to prevent sleep
- OR use Railway instead (no sleep)

---

### **2. Custom Domain (Optional)**

**Vercel:**
1. Go to Project Settings → Domains
2. Add your domain (e.g., `transitops.yourdomain.com`)
3. Update DNS records as shown
4. SSL auto-configures!

**Render:**
1. Go to Service → Settings → Custom Domains
2. Add domain
3. Update DNS
4. SSL auto-configures!

## 🆘 Troubleshooting

### **Login fails / CORS error**
1. Check `TRANSITOPS_CORS_ORIGINS` in Render includes your Vercel URL
2. No trailing slashes in URLs
3. Wait for backend to finish redeploying

### **Backend sleeping**
- First request takes 30-60 seconds
- Normal for free tier
- Upgrade to Starter plan ($7/mo) to fix

### **Database empty**
- Run seed script again in Render Shell:
  ```bash
  cd transitops-backend
  python seed.py
  ```

### **Build fails**
- Check build logs in platform dashboard
- Verify all files are committed and pushed
- Check `requirements.txt` and `package.json`
